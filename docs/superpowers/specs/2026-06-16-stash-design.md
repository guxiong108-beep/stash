# Stash — 设计文档

- 日期：2026-06-16
- 状态：待实现（设计已确认）
- 平台：Windows 优先，未来兼顾 macOS

## 1. 定位

Stash 是一个轻量级桌面工具，对标 Raycast，面向中文用户。通过全局快捷键唤起一个统一搜索栏，可以：

- 搜索并打开**应用**
- 搜索并打开**最近打开过的文件**
- 浏览与复用**剪贴板历史**

招牌能力：在剪贴板历史里**勾选多条记录形成队列**，之后用专属快捷键**逐条依次粘贴**到目标程序。所有数据本地持久化，关机重启后历史不丢。

## 2. 核心决策（已确认）

| 主题 | 决定 |

## 3. 技术架构

Tauri 应用，分两层：

- **Rust 后端（core）**：全局快捷键、剪贴板监听、系统剪贴板读写、按键注入、应用枚举、最近文件读取、SQLite 持久化、图片存储、托盘、开机自启。
- **Web 前端（UI）**：命令栏浮层、模式标签、结果列表、剪贴板历史（多选）、设置页、双主题。

**IPC**：前端经 Tauri command 调用后端能力；后端经 event 主动推送（剪贴板新增条目、队列粘贴进度等）。

## 4. 模块拆分

每个模块单一职责、接口清晰、可独立测试。

| 模块 | 职责 | 主要依赖 |
|------|------|----------|
| 命令栏窗口（前端） | 搜索框 + 模式标签 + 结果列表，键盘导航（↑↓ 选择、↵ 打开、Tab 切标签） | 搜索聚合器（IPC） |
| 剪贴板监听（Rust） | 监听系统剪贴板变化、去重、写库、强制 200 条上限、跳过敏感内容 | 剪贴板存储 |
| 剪贴板存储（Rust + SQLite） | 条目表（文本/图片元数据）、图片落盘 + 缩略图、置顶标记、按日期查询 | SQLite、文件系统 |
| 粘贴队列引擎（Rust） | 持有勾选的有序条目；`Alt+V` 弹出下一条注入光标处；浮窗显示「剩 N 条 / 下一条」 | 按键注入、浮窗 |
| 应用索引（Rust） | 扫描开始菜单 .lnk + 已安装程序，按使用频率排序 | 文件系统、注册表 |
| 最近文件（Rust） | 读 `%APPDATA%\Microsoft\Windows\Recent`，解析 .lnk 取目标，取最近 50；打开文件 / 打开所在文件夹 | 文件系统 |
| 搜索聚合器（Rust） | 按当前标签把查询分发到对应数据源并合并/排序 | 应用索引、最近文件、剪贴板存储 |
| 系统集成（Rust） | 全局快捷键注册、系统托盘、开机自启 | Tauri 插件 |
| 设置（前端 + Rust） | 快捷键、主题、保留条数（免费锁 200）、清空历史，持久化到 config.json | 配置存储 |
| 主题（前端） | 浅色 / 暖色，默认暖色 | — |

## 5. 关键数据流

### 5.1 复制时
1. 系统剪贴板变化 → 剪贴板监听捕获
2. 敏感来源检测 → 命中则跳过
3. 去重检查（与最新条目相同则跳过/提到最前）
4. 若为图片 → 原图落盘 `images/` + 生成缩略图 `thumbs/`
5. 写入 SQLite（带时间戳）
6. 超过 200 条 → 淘汰最旧（置顶项不淘汰）
7. 推送 event → UI 刷新（按日期，最新在上）

### 5.2 队列粘贴
1. 剪贴板模式下勾选多条（保持点选顺序）
2. 确认 → 粘贴队列引擎建队列，焦点回到目标程序
3. 每按 `Alt+V` → 取队首一条注入当前光标处 → 更新浮窗进度
4. 队列排空 → 浮窗自动关闭

## 6. 存储设计

`%APPDATA%\Stash\`：

- `stash.db` — SQLite，存剪贴板文本与元数据、置顶状态、时间戳；应用/最近文件的使用频率
- `images/` — 剪贴板图片原图
- `thumbs/` — 图片缩略图
- `config.json` — 设置（快捷键、主题、保留条数等）

剪贴板条目表核心字段：`id`、`type`（text/image）、`content`（文本）或 `image_path`/`thumb_path`、`created_at`、`pinned`、`source_app`（用于敏感过滤/展示）。查询默认 `ORDER BY pinned DESC, created_at DESC`。

## 7. 错误处理

| 场景 | 处理 |
|------|------|
| 剪贴板读取失败/格式不支持 | 跳过并记日志，不打断监听 |
| Recent 目录缺失/无权限 | 该模式显示空列表 + 友好提示 |
| 全局快捷键被占用 | 设置页标红冲突项，提示用户改键 |
| 图片落盘失败 | 仅保留文本部分（若有），记日志 |
| 注入时无焦点输入框 | 浮窗提示「请先点选目标输入框」 |
| 超上限淘汰 | 仅淘汰非置顶的最旧项 |

## 8. 测试策略

- **Rust 单元测试**：去重逻辑、200 条淘汰（含置顶豁免）、队列出队顺序、.lnk 解析、配置读写、按日期排序查询。
- **前端组件测试**：命令栏过滤、标签切换、多选交互、主题切换、键盘导航。
- **手动 E2E**：全局快捷键、按键注入、托盘、开机自启（依赖真实系统，难自动化）。

## 10. 待解决/已知风险

- Windows 按键注入在部分受 UAC 提权的窗口可能受限（需实测，必要时回退到「写入系统剪贴板 + 提示手动粘贴」）。
- 已安装程序枚举在不同 Windows 版本下来源不一（开始菜单 .lnk 较稳，注册表卸载项作补充）。
- 敏感内容识别目前依赖来源应用判定，无法 100% 覆盖；以「默认不记录已知密码管理器 + 一键清空」兜底。
